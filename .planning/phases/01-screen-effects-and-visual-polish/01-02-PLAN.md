  <action>
Add a `ScreenFade` IIFE namespace in a new `<script>` block BEFORE the main game script.

**ScreenFade API:**
- `ScreenFade.start(type, duration, callback)` — starts a fade effect
  - `type`: `'out'` (screen goes black), `'in'` (black clears to transparent), `'out-in'` (fade to black, execute callback at midpoint, then fade back in)
  - `duration`: total duration in dt units (e.g., 30 = ~0.5 seconds at 60fps)
  - `callback`: function called at the midpoint of 'out-in' (used to actually switch game state and reset game)
- `ScreenFade.update(dt)` — advance fade timer, call callback at midpoint for 'out-in' type
- `ScreenFade.draw(ctx, w, h)` — draw black fullscreen rectangle with current alpha. Draw AFTER everything else in render (last thing drawn)
- `ScreenFade.active` — boolean, true while a fade is in progress

**State:**
- `alpha`: 0-1 current fade darkness
- `timer`: countdown
- `phase`: 'fading-out' or 'fading-in' (for out-in transitions)
- `midCallback`: the callback for out-in transitions
- `totalDuration`: stored for calculating alpha

**Transitions to add (wire into main game script):**

1. **Menu to Playing (line ~1238, ~768):** When player clicks "JUCA!" or presses Space to start:
   - Replace immediate `gameState = 'playing'; resetGame();` with:
   - `ScreenFade.start('out-in', 30, () => { gameState = 'playing'; resetGame(); })`
   - During fade-out, menu is still visible; at midpoint game resets; during fade-in, gameplay appears

2. **Playing to GameOver (line ~1452, ~1638, ~1683):** When collision kills player:
   - Keep `gameState = 'gameover'` immediate (game needs to stop updating physics)
   - Add `ScreenFade.start('in', 20)` to fade FROM a flash-like darker overlay back to clear — gives a brief visual "impact" moment before game-over screen appears
   - Alternative: use a quick white or red flash that fades: set alpha to 0.4 immediately then decay. This is more "impact" feeling than a slow fade.

3. **GameOver to Playing (line ~1218):** When player restarts after game over:
   - Replace immediate restart with `ScreenFade.start('out-in', 25, () => { gameState = 'playing'; resetGame(); })`

**Integration in render():**
After the very last draw call in `render()` (after `drawGameOverScreen()`), add:
```javascript
ScreenFade.draw(ctx, canvas.width, canvas.height);
```

**Integration in update():**
At the top of `update(dt)`, add:
```javascript
ScreenFade.update(dt);
```

**Input blocking during fade:**
When `ScreenFade.active` is true, block game input (space/click to jump, click to restart). Add an early return in the input handlers:
```javascript
if (ScreenFade.active) return;
```
Keep fade duration short (20-30 dt units = 0.3-0.5 seconds) so players never feel stuck.
