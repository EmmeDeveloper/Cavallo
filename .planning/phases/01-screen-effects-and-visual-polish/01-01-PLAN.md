<task type="auto">
  <name>Task 1: Create ParallaxBg subsystem with procedural layers</name>
  <files>index.html</files>
  <action>
Add a new `<script>` block BEFORE the main game `<script>` in index.html. Create a `ParallaxBg` IIFE namespace with the following:

**Layer structure (3 layers):**
1. **Far layer (speed 0.15):** Procedural sky gradient — dark warm tones at top (#2a0a1a) fading to dusky orange (#cc6040) at bottom. This scrolls very slowly to simulate distant atmosphere. Draw as a full-canvas-height gradient. No image needed.
2. **Mid layer (speed 0.4):** Procedural city silhouette — draw simple rectangular building shapes in dark tones (#1a0a0a to #2a1515) along the bottom 40% of the background area (from bgH * 0.6 to bgH). Randomize building widths (20-60px) and heights (40-120px) at init to create variety. Store building data in an array generated once. Draw buildings as filled rectangles scrolling left. When a building scrolls fully off-screen left, wrap it to the right side.
3. **Near layer (speed 1.0):** The existing bg.webp image, drawn at current scroll speed (same as ground). This preserves what already works.

**API:**
- `ParallaxBg.init(canvasWidth, bgHeight)` — generate building array for mid layer, store dimensions
- `ParallaxBg.update(dt, speed)` — advance each layer's x offset by `speed * layer.speed * dt`, wrap seamlessly using modulo
- `ParallaxBg.draw(ctx, canvasWidth, bgHeight)` — draw all 3 layers in order (far, mid, near)

**Critical constraints (from PITFALLS.md):**
- Use `Math.floor()` on all x/y coordinates passed to `drawImage()` and `fillRect()` to avoid sub-pixel blur
- The near layer bg.webp must use the seamless wrap technique: draw twice side by side (`x` and `x + canvasWidth`) and wrap via modulo
- Do NOT use `shadowBlur` on any layer — it kills performance
- Far gradient can be drawn once to an offscreen canvas at init and reused via drawImage for performance (avoid recreating gradient every frame)

**Mid-layer building generation:**
Generate 30-40 buildings at init with randomized widths and heights. Store as `{ x, w, h }`. Total width should be ~2x canvas width to ensure seamless wrap. Buildings scroll at 0.4x speed. When the entire building strip has scrolled left by its total width, reset offset to 0.
