**CSS changes (in the existing style block):**

2. In the `canvas` rule: remove `max-width: 100vw;` and `max-height: 100vh;` — JS will control size
3. In the `canvas` rule: add `display: block;` — prevents inline baseline gap

**JS changes (in the main game script, immediately after line 543 `ParallaxBg.init(canvas.width, 540);`):**

Add the resizeCanvas function and event listeners:

```javascript
// ==================== RESPONSIVE CANVAS SCALING ====================
function resizeCanvas() {
  const nativeW = canvas.width  || 960;
  const nativeH = canvas.height || 590;
  const scale   = Math.min(window.innerWidth / nativeW, window.innerHeight / nativeH);
  canvas.style.width  = Math.floor(nativeW * scale) + 'px';
  canvas.style.height = Math.floor(nativeH * scale) + 'px';
}
resizeCanvas();
window.addEventListener('resize', resizeCanvas);
if (screen.orientation) {
  screen.orientation.addEventListener('change', resizeCanvas);
}
```

Then add `resizeCanvas();` as the LAST line inside `bgImage.onload` (after `ParallaxBg.init(canvas.width, bgH);` at line 536) — this ensures the canvas display size updates if bgImage changes canvas.height.

**CRITICAL — do NOT:**
- Change `canvas.width` or `canvas.height` in resizeCanvas — only change `.style.width` and `.style.height`
- Modify `getCanvasPos()` — it already handles CSS scaling correctly via getBoundingClientRect
- Touch any HUD drawing code — all HUD positions use canvas.width as anchor and remain correct
- Use `window.orientationchange` (deprecated) — use `screen.orientation.addEventListener('change', ...)` with the `if (screen.orientation)` guard
