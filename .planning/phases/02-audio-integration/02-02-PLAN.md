**Step C — Add mute button to canvas HUD.**

1. Declare `btnMute` rect at file scope, near `btnHome` declaration (line ~1435):
   ```javascript
   let btnMute = { x: 0, y: 0, w: 0, h: 0 };
   ```

2. In `drawScore()` function, AFTER the btnHome drawing block (after the door drawing, around line ~2079), add the mute button:
   ```javascript
   // Mute button (left of home button)
   const mx = hx - 42, my = 8, mw = 32, mh = 32;
   btnMute = { x: mx, y: my, w: mw, h: mh };
   ctx.fillStyle = 'rgba(0,0,0,0.5)';
   ctx.fillRect(mx, my, mw, mh);
   ctx.strokeStyle = AudioManager.isMuted() ? '#555' : '#ff6633';
   ctx.lineWidth = 1;
   ctx.strokeRect(mx, my, mw, mh);
   // Simple speaker icon: 'S' when unmuted, 'M' when muted (consistent pixel-art look)
   outText(AudioManager.isMuted() ? 'M' : 'S',
     mx + mw / 2, my + mh / 2 + 5,
     AudioManager.isMuted() ? '#666' : '#ddd',
     'bold 14px "Courier New", monospace', 'center');
   ```

   Use 'S' (sound) and 'M' (muted) text labels per RESEARCH.md recommendation — avoids cross-platform emoji rendering issues.

3. In `handleGameClick()` (line ~1485), add mute button hit test BEFORE the existing btnHome check:
   ```javascript
   if ((gameState === 'playing' || gameState === 'gameover') && hitTest(pos, btnMute)) {
     AudioManager.setMute(!AudioManager.isMuted());
     return true;
   }
   ```

   Also add the same hit test in the touch/pointer handler if there is a separate one that handles gameplay touches. Check for any `canvas.addEventListener('pointerdown'` or `canvas.addEventListener('touchstart'` handler that might bypass `handleGameClick`.

ANTI-PATTERNS TO AVOID:

- Do NOT use emoji for the mute icon — use text 'S'/'M' for consistent rendering
- Do NOT add a second `AudioManager.init()` call — it is already called once (from plan 01)
